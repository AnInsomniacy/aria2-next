# Documentation

This directory contains user, contributor, release, and maintenance documentation for aria2-next.

## Contributor and Community Documents

| Path | Purpose |
| --- | --- |
| `CONTRIBUTING.md` | Local setup, source rules, testing policy, dependency policy, PR expectations, and release-change rules |
| `SUPPORT.md` | Where to ask questions, report bugs, request features, report packaging problems, and report vulnerabilities |
| `SECURITY.md` | Private vulnerability reporting process and supported security scope |
| `PRIVACY.md` | Local data, network behavior, RPC exposure, and public-log redaction guidance |
| `CODE_OF_CONDUCT.md` | Conduct expectations and enforcement scope |
| `RELEASE_INTEGRITY.md` | Official artifact naming, checksum verification, signing status, and failed-release policy |
| `RELEASE.md` | Maintainer release flow, workflow trigger behavior, packaging guarantees, and recovery rules |
| `TROUBLESHOOTING.md` | Common engine failures and the evidence needed for useful reports |
| `INTEGRATION.md` | Stable executable and JSON-RPC integration guidance for applications and scripts |

## GitHub Community Files

| Path | Purpose |
| --- | --- |
| `.github/ISSUE_TEMPLATE/bug_report.yml` | Reproducible engine bug intake |
| `.github/ISSUE_TEMPLATE/crash_report.yml` | Process crash, abort, WER, core dump, and debugger evidence intake |
| `.github/ISSUE_TEMPLATE/build_packaging.yml` | Local build, CI, release artifact, dependency, and packaging failure intake |
| `.github/ISSUE_TEMPLATE/feature_request.yml` | Focused engine feature proposal intake |
| `.github/ISSUE_TEMPLATE/usage_question.yml` | Usage, configuration, and integration question intake |
| `.github/PULL_REQUEST_TEMPLATE.md` | PR scope, compatibility, verification, release-note, and AI-disclosure checklist |

## Technical Documents

| Path | Purpose |
| --- | --- |
| `manual/` | Sphinx manual sources for CLI, RPC, options, and technical notes |
| `completion/` | Bash completion source and generation tooling |
| `maintenance/` | Durable audit records and long-running modernization trackers |
| `licenses/` | Preserved license exception and dependency license notes |
| `media/` | Repository media assets used by documentation |

Generated reports, scratch logs, packet captures, release staging folders, and temporary downloads do not belong in this directory.
